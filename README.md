## 1. Reconeix la utilitat de les aplicacions d'ofimàtica web
 
1. **Explica per què GitHub es pot considerar una aplicació web de treball col·laboratiu.**
 
  GitHub és una plataforma que esta al núvol ja que no cal instalar res, simplement entres desde el navegador i ja esta. Diferents persones poden treballar al mateix codi encara que estiguin en llocs diferents i fins i tot al mateix temps. També te eines com issues o pull requests que ajuden bastant a organitzar-se, és a dir, una mica com un Google Docs pero per codi.
 
2. **Quins avantatges té utilitzar GitHub en lloc de guardar els fitxers d'un projecte només en un ordinador local?**
 
  Si nomes ho tens al teu ordinador i es trenca o el perds doncs ja esta ho perds tot. En canvi amb GitHub sempre ho tens guardat al núvol i pots accedir desde qualsevol lloc. També si treballes amb gent no cal anar passant arxius tot el rato, tothom treballa al mateix lloc. I si algu fa algo malament al codi pots tornar enrere sense massa problema.
 
3. **Com pot ajudar GitHub a gestionar versions d'un projecte web o d'una aplicació?**
 
  Cada commit és com fer una foto del projecte en aquell moment. Això fa que tinguis tot l'historial de canvis, qui ha fet que i quan. Si alguna cosa deixa de funcionar pots mirar versions anteriors i veure on esta el problema exactament.
 
4. **En quins tipus de projectes és especialment útil utilitzar Git i GitHub?**
 
  Sobretot en projectes de programacio encara que en general serveix per bastants coses. Va molt be quan hi ha mes d'una persona treballant, per exemple apps, webs, videojocs o fins i tot treballs de classe. També en projectes llargs ja que el codi va canviant molt.
 
5. **Quines diferències hi ha entre un sistema de control de versions com Git i un sistema tradicional d'emmagatzematge de fitxers?**
 
  Amb una carpeta normal nomes tens la versio actual i ja esta. Amb Git tens tot l'historial de canvis des del principi i pots veure com ha anat evolucionant. També permet treballar en branques separades sense molestar als altres cosa que amb carpetes compartides es bastant complicat.

## 2. Classifica segons la funcionalitat i prestacions específiques
 
6. **Explica quina és la diferència entre Git i GitHub.**
 
  Git és el programa que tens al ordinador i controla els canvis del codi. GitHub és la web on puges el repositori per compartir-lo i tenir-lo al núvol. És a dir Git seria el motor i GitHub el cotxe.
 
7. **Quines funcionalitats ofereix GitHub a més del control de versions del codi?**
 
  A part de guardar codi també te altres coses útils com issues per tasques o errors, pull requests per revisar codi, GitHub Pages per fer webs gratis, un wiki i també estadistiques del projecte.
 
8. **Esbrina i defineix les següents funcionalitats de GitHub segons el seu ús:**
 
  - **Repositoris:** És com una carpeta al núvol on tens tot el projecte amb els fitxers i historial.
 
  - **Issues:** Serveixen per apuntar errors o coses a fer, l'equip pot comentar i organitzar-ho.
 
  - **Pull requests:** Serveixen per proposar canvis i que els altres els revisin abans d'afegir-los.
 
  - **GitHub Pages:** Permet pujar una web directament desde el repositori gratis.
 
9. **Quina funció tenen les **branques (branches)** en Git?**
 
  Permeten treballar en parts diferents del projecte sense tocar la principal. Cada persona pot fer la seva part i després unir-ho tot.
 
10. **Per a què serveixen les **pull requests** en un projecte col·laboratiu?**
 
   Serveixen per revisar els canvis abans de posar-los al projecte principal. Així evites errors i el codi dolent.

## 3. Gestiona els comptes d'usuari
 
11. **Què és un compte d'usuari a GitHub i per què és necessari per treballar amb repositoris remots?**
 
   És el teu perfil a GitHub i sense ell no pots fer gaire cosa com pujar codi o col·laborar. També serveix per saber qui fa cada canvi.
 
12. **Quina diferència hi ha entre un repositori **públic** i un **privat** a GitHub?**
 
   Un public el pot veure tothom encara que no tingui compte. Un privat nomes el veuen els que estan convidats.
 
13. **Com es pot afegir un col·laborador a un repositori de GitHub?**
 
   Vas a settings, collaborators i afegeixes l'usuari. Ell accepta i ja pot treballar.
 
14. **Quins rols o permisos pot tenir un usuari dins d'un repositori?**
 
   Hi ha diferents nivells com read, write o admin. Depen del que pugui fer cada usuari.
 
15. **Per què és important gestionar correctament els permisos dels usuaris en un projecte col·laboratiu?**
 
   Per evitar que algu faci canvis que no toca o esborri coses per error.
